# 贡献指南

感谢您对本项目的贡献！我们欢迎各种形式的贡献，包括代码改进、文档完善、bug报告和新功能建议。

## 🌟 贡献类型

### 1. 代码贡献
- **功能改进**：优化现有算法或添加新功能
- **性能优化**：提升代码运行效率
- **bug修复**：修复发现的问题
- **测试用例**：添加测试代码确保稳定性

### 2. 文档贡献
- **文档完善**：改进README、代码注释
- **使用示例**：添加更多使用案例
- **教程编写**：编写使用教程和最佳实践

### 3. 问题反馈
- **bug报告**：详细描述遇到的问题
- **功能建议**：提出新功能或改进建议
- **性能问题**：报告性能相关的问题

## 🚀 快速开始

### 环境设置
```bash
# 克隆仓库
git clone https://github.com/your-username/hybrid-retrieval-evaluation.git
cd hybrid-retrieval-evaluation

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt
```

### 开发流程
1. **Fork仓库**到您的GitHub账户
2. **创建功能分支**：`git checkout -b feature/your-feature-name`
3. **进行更改**：添加您的改进或修复
4. **测试更改**：确保所有测试通过
5. **提交更改**：`git commit -m "Add: 描述您的更改"`
6. **推送分支**：`git push origin feature/your-feature-name`
7. **创建Pull Request**：在GitHub上提交PR

## 📋 贡献规范

### 代码规范
- **遵循PEP8**：Python代码应符合PEP8规范
- **类型注解**：为新函数添加类型提示
- **文档字符串**：为函数和类添加docstring
- **注释清晰**：复杂逻辑需要详细注释

### 提交信息规范
- **格式**：`类型: 简要描述`
- **类型**：
  - `Add`: 添加新功能
  - `Fix`: 修复bug
  - `Update`: 更新现有功能
  - `Docs`: 文档更新
  - `Test`: 测试相关
  - `Style`: 代码格式调整

示例：
```
Add: 实现自适应权重优化算法
Fix: 修复互信息计算的数值稳定性问题
Update: 改进实验结果可视化
```

### 分支命名规范
- `feature/功能名称`：新功能开发
- `fix/问题描述`：bug修复
- `docs/文档内容`：文档更新
- `test/测试内容`：测试相关

## 🧪 测试要求

### 单元测试
- 为新功能添加对应的测试用例
- 确保所有测试通过：`python -m pytest tests/`
- 测试覆盖率应达到80%以上

### 集成测试
- 测试主要功能模块的集成
- 验证端到端的功能正确性
- 性能测试确保算法效率
